Создание окна

3. Создание обработчика событий

Обработчик сообщений (событий) – это цикл, в котором расположена функция MainWinProc(), занимающаяся непосредственной обработкой сообщений.

Задание 11
Запишите после регистрации окна (внутри операторных скобок функции WinMain()) следующее:

MSG msg;  //переменная хранящая сообщения окна
ShowWindow( hwnd, SW_SHOWDEFAULT );  //нарисуем окно
UpdateWindow( hwnd );  //обновим окно
ZeroMemory( &msg, sizeof(msg));
while( msg.message!=WM_QUIT)  //если пришедшее сообщение не выход, то выполняем итерацию
{
  if(PeekMessage( &msg, NULL,0,0,PM_REMOVE ))  //если есть сообщение, то обрабатываем его, а после удаляем из очереди
  {
    TranslateMessage( &msg );  //преобразование сообщения связанных с клавишами
    DispatchMessage( &msg );  //пересылка сообщений в MainWinProc
  }
  else  //если нет сообщения, то пока ничего не делаем
  {};
}

Функция MainWinProc() занимается непосредственной обработкой сообщений. Те сообщения, которые она не обработает, будет обрабатывать ОС.
Вот прототип этой функции:

LRESULT CALLBACK MainWinProc
(
HWND hwnd,  //дискриптор окна
UINT msg,  //сообщение windows
WPARAM wparam,  //дополнительные параметры
LPARAM lparam  //дополнительные параметры
)

Задание 13
Внутри финуции (в операторных скобках) MainWinProc() напишите вот такой код:

switch(msg)
{
  case WM_DESTROY:  //если пришло сообщение закрытия окна, то
  {
    PostQuitMessage(0);  //закрываем его
    return(0);  //заканчиваем выполнение функции MainWinProc()
  }
}

Но вы, конечно, не будете перечислять все возможные сообщения, которые могут "прилететь". Поэтому воспользуйтесь функцией DefWindowProc. Эта функция заставит ОС обработать сообщение окна по умолчанию.

Задание 14
Запишите предложенный ниже код после оператора switch в функции MainWinProc():

return DefWindowProc(hwnd, msg, wparam, lparam);

Вот и все. Возможно вам это показалось немного запутанным. Не бойтесь! Со временем придет и понимание.
