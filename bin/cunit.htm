Анимация

Класс CUNIT

Класс CUNIT позволяет работать с морфирующей анимацией. Вот его описание:

class CUNIT
{
  public:
  //Контейнер для хранения данных о модели
  CANIMATION Mesh;
  //Текстура
  LPDIRECT3DTEXTURE9 Texture1;
  //Устройство главного объекта
  LPDIRECT3DDEVICE9 D3DDevice;
  //Буфер вершин
  LPDIRECT3DVERTEXBUFFER9 BufferVershin;
  
  //Уголы нормали плоскости, на которой находиться модель в радианах
  float cos_a;
  float cos_b;
  float cos_c;
  //Смещешие объекта относительно мировой матрицы
  CUSTOMVERTEX Place;
  //Угол поворота вокруг Z в градусах
  float rotate;
  //Размер
  float scale;
  //Смещение (шаг в пикселях)
  float speed;

  //Режим работы со светом
  _RRFLAG rrlight;
  //Позиция анимации
  float a_position;
  //Добавка к позиции анимации
  float a_add;
  //Номер анимации
  unsigned short int a_nom;

  public:
  CUNIT();
  //Уничтожает D3DDevice, BufferVershin, BufferIndex, Texture1
  void Release();
  //Отображение загруженной модели на устройстве по ссылке D3DDevice
  HRESULT Present();
  //Переписываем буфера
  HRESULT ReBuf(CUSTOMVERTEX* vertex);

  //Создание динамической модели
  HRESULT CreateModelFF(char passfile[], LPDIRECT3DDEVICE9 Device);
  //Получение меша и перезапись буферов
  HRESULT Play();

  //Расчет гравитации
  bool Gravity(CUSTOMVERTEX* Vertex, unsigned short int kol_vertex);

  //Расчет движения на основе команды act и параметра ang (угол)
  void DoAction(_UACTION act, float ang);

  //Смена анимации
  void SetAnimation(unsigned short int nom_anim);
};

Информация о меше
Информация о морфинге храниться в поле Mesh типа CANIMATION класса CUNIT. CANIMATION - это класс, описанный в CAnimation.h и CAnimation.cpp, позволяющий загружать, хранить и вычислять меш. Он наследуется классом CUNIT агрегативным методом.

Номер анимации - есть поле unsigned short int a_nom класса CUNIT, которое изначально равно 0.
Позиция в анимации есть поле float a_position, обозначающий завершенность анимации в процентах. Может принимать значения от 0 до 1. Изначально установлен 0.
Величина, на которую будет происходить изменение позиции (a_position), есть поле float a_add, изначально равное 0.01.

Для вычисления очередного кадра в соответствии с полями a_nom, a_position, a_add используется метод Play(), который при вызове увеличит поле a_position на величину a_add (a_position+=a_add) и вызовет метод this->Mesh.GetMorphedMesh (a_nom, a_position).
В результате своей работы метод Play() заполнит буфер вершин по ссылке BufferVershin.

Для того чтобы информацию из вершинных буферов передать на объект видеокарты, который располагается по ссылке D3DDevice, нужно воспользоваться методом Present().

Для смены анимации вызывается метод SetAnimation класса CUNIT. Он имеет 1 параметр - номер новой анимации. После вызова данного метода будет изменено поле a_nom, а так же поле a_position станет равным a_position=0-a_add.
Так же метод SetAnimation позволяет реализовать технологию связующего меша, которая заключается в том, что последний результирующий меш предыдущей анимации становиться первым мешем следующей анимации. Данная технология позволяет сгладить переходы от одной анимации к другой. В результате, перетекание мешей друг в друга осуществляется не только в районе цепочки одной анимации, но и между самими цепочками анимаций!

Пространственное расположение меша
За пространственное расположение меша отвечают следующие поля класса CUNIT:
- cos_a, cos_b, cos_c - это углы (в радианах) нормали поверхности, на которой находиться модель. Локальная система координат выравнивается таким образом, чтобы ось Z (локальная) была параллельна этой нормали.
- rotate - это угол поворота вокруг локальной оси Z (в градусах).
- Place - это место расположения модели, т.е. точки начала локальных координат. В данном использовании этого класса (CUSTOMVERTEX) используются только поля x, y, z. Поля nx, ny, nz, tu, tv не используются. Вы можете использовать их по своему усмотрению.

Гравитация
Расчет гравитации (смещения модели по мировой оси Z в отрицательную сторону) производиться по средствам функции Gravity, имеющей 2 параметра. Первый параметр CUSTOMVERTEX* Vertex - это массив вершин (образующих треугольники) с которыми обсчитывается столкновение. Обещает столкновения - это так же одна из задач данной функции. Второй параметр - это количество вершин, которые передаются в первом параметре.
Давайте поговорим конкретнее о работе данной функции. Вначале функция проверяет нет ли столкновения модели (точки начала локальной системы координат) с треугольниками из массива переданного по первому входному параметру. Если такое столкновение присутствует, то гравитацию не обсчитываем и модель как бы застревает на этом полигоне, причем поля cos_a, cos_b, cos_c автоматически становятся равные таким значениям, при которых ось Z локальной системы координат будет параллельна нормали первой вершины треугольника, на котором застряла модель.
У этой функции есть много минусов. Так что я предлагая вам самим её переопределить. Но это в будущем, а сейчас давайте подведем итог по данной функции.
Правила использования метода Gravity:
- Данный метод следует вызывать после всевозможных позиционирований модели в пространстве;
- Данный метод следует вызывать до метода Present;
- Если вы имитируете движение какого-либо объекта и его столкновение с препятствием то не забывайте, что перемещение этого объекта более чем на 10 пикселей за один вызов метода Gravity может привести к проскакиванию модели через препятствие.
- Важно помнить, что изменение полей cos_a, cos_b, cos_c вычисляется по нормали первой вершины треугольника, на котором "застряла" модель.

Выполнение движения
Метод DoAction позволяет на основе двух входных параметров: команды (_UACTION act) и угла (float ang) выполнять различные действия. Тип _UACTION описан в Types.cpp . Данный метод принимает всего два значения данного типа:
- U_MOVE движение в направлении ang со скоростью указанной в поле speed (длинна шага в px) класса CUNIT. Если, к примеру, в ang записать поле rotate класса CUNIT, то модель начнет двигаться в направлении заданном в этом поле. В результате будет автоматически изменятся поле Place класса CUNIT.
- U_TURN поворот модели так, чтобы rotate был равен ang. Причем rotate изменяется не сразу, а постепенно на 1 градус при очередном вызове метода DoAction. Это дает возможность плавно поворачивать модель в течение времени.

Другая информация о классе
HRESULT ReBuf(); - метод, отвечающий за перезапись вершинного буфера. Данный метод используется, к примеру, в том случае, когда хранилище меша (Mesh) был заполнен не с помощью функции CreateModelFF, а каким-нибудь иным способом. Но следует помнить, что буфер вершин должен быть создан и указатель на него должен быть размещен в BufferVershin.
void Release(); - метод, уничтожает буфера Texture1, BufferVershin.
_RRFLAG rrlight; - перечисляемый тип (Флаг режима работы), описанный в файле Types.cpp . В данном случае используется только два значения: RR_LIGHTON - вкл. обработку света для данной модели, RR_LIGHTOFF - выкл. обработку света для данной модели.

Класс CUNIT, как видите очень большой, и сложный. Попробуйте воспользоваться некоторыми изученными выше функциями.
Смена анимации очень полезная вещь. Как уже писалось выше, за это действие отвечает метод SetAnimation.

Задание 8
Внутри функции MainWinProc используется оператор switch для выбора реакции приложения на пришедшее событие. Вам сюда.
Идея такая: сейчас вы сделаете так, чтобы при нажатии на пробел ваш паук менял одну анимацию на другую, и так по кругу. Для этого добавьте в операторные скобки switch дополнительное условие:

case WM_KEYDOWN:
{
  //тут будем обрабатывать нажатые кнопки

  break;
};

Этот код можно интерпретировать так: если какая-то кнопка нажата до делаете то, что в операторных скобках.
Но кнопок на клавиатуре много! Информация о том, какая кнопка нажата храниться в переменной wparam, которая приходит вместе с сообщением от ОС нашему приложению.
